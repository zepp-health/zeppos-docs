## Register Page

For each page in the Mini Program, you need to use [`Page` constructor](../../../reference/device-app-api/newAPI/global/Page.mdx) in the corresponding js file of the page to register the page instance, specify the page lifecycle callbacks, event handling functions, and can mount properties on the page instance.

:::caution
For each page in the Mini Program, you need to configure the path in the `targets` object of `app.json`.
:::

Use the [`getCurrentPage` method](../../../reference/device-app-api/newAPI/global/getCurrentPage.mdx) to get the `page` instance.

```js title="page.js"
Page({
  state: {
    text: 'Hello Zepp OS'
  },
  onInit() {
    console.log('onInit')
  },
  build() {
    console.log('build')
    console.log(this.state.text)
  }
})

const page = getCurrentPage()
console.log(page._options.state.text)
```

## Draw widgets

Zepp OS provides rich UI widgets.

The entire widget layout uses absolute coordinates, and most widgets have the following properties.

| Property | Description     |
| ---- | -------- |
| x    | X coordinate   |
| y    | Y coordinate   |
| w    | Widget width |
| h    | Widget height |

The coordinates of the widgets in the document are absolute coordinates starting from the screen axis unless otherwise specified.
